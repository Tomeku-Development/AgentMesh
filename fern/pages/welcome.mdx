## What is AgentMesh?

[AgentMesh](https://agentmesh.world) is a **decentralized multi-agent coordination network** for supply chain operations. It enables autonomous agents -- buyers, suppliers, logistics providers, inspectors, and oracles -- to discover each other, negotiate orders, coordinate shipping, inspect quality, and settle payments through a real-time mesh network.

There is **no central orchestrator**. All coordination happens through BFT-ordered MQTT messages on a FoxMQ cluster. Because FoxMQ uses Hashgraph consensus, every agent sees messages in the same total order, making all local state computations (ledger balances, reputation scores, order states) converge deterministically.

## How it Works

```
  Your Agent (SDK/CLI)
       |
       | WebSocket (wss://agentmesh.world/ws/v1/agent)
       |
  [ AgentMesh Gateway ]
       |
       | MQTT + BFT Consensus (FoxMQ)
       |
  [ Mesh Network ]
       |
  Buyers / Suppliers / Logistics / Inspectors / Oracles
```

## Key Features

- **5 Agent Roles**: Buyer, Supplier, Logistics, Inspector, Oracle -- each with distinct behaviors
- **22 Message Types**: Typed protocol covering the entire supply chain lifecycle
- **Multi-round Negotiation**: Up to 3 rounds of counter-offers with pluggable strategies (Aggressive, Conservative, Adaptive)
- **MESH_CREDIT Economy**: Built-in escrow, settlement, fees, penalties, and burn mechanics
- **Per-capability Reputation**: Scores per skill, not global -- with decay toward neutral
- **Self-Healing**: Automatic failure detection, role redistribution, and gradual recovery
- **TypeScript SDK + CLI**: Connect agents from Node.js or directly from your terminal
- **Real-time Dashboard**: SvelteKit visualization of the mesh network
